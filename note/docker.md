
## ১. গুরুত্বপূর্ণ ডকার কমান্ডসমূহ

### **ইমেজ (Image) সম্পর্কিত কমান্ড**

| কমান্ড | কেন ব্যবহার করা হয় |
| --- | --- |
| `docker build -t image_name .` | বর্তমান ডিরেক্টরির `Dockerfile` থেকে একটি ইমেজ তৈরি করতে। |
| `docker images` | আপনার পিসিতে থাকা সব ডকার ইমেজের লিস্ট দেখতে। |
| `docker rmi image_id` | কোনো নির্দিষ্ট ইমেজ ডিলিট বা রিমুভ করতে। |
| `docker pull image_name` | Docker Hub থেকে কোনো ইমেজ ডাউনলোড করতে (যেমন: `docker pull node`)। |

### **কন্টেইনার (Container) সম্পর্কিত কমান্ড**

| কমান্ড | কেন ব্যবহার করা হয় |
| --- | --- |
| `docker run -d -p 5000:3000 image_name` | ইমেজ থেকে কন্টেইনার তৈরি করে ব্যাকগ্রাউন্ডে রান করতে। |
| `docker ps` | বর্তমানে কোন কোন কন্টেইনার চলছে তা দেখতে। |
| `docker ps -a` | বন্ধ বা চালু সব কন্টেইনারের লিস্ট দেখতে। |
| `docker stop container_id` | একটি চলমান কন্টেইনার বন্ধ করতে। |
| `docker rm container_id` | একটি বন্ধ কন্টেইনার ডিলিট করতে। |
| `docker logs -f container_id` | কন্টেইনারের ভেতরে কী ঘটছে বা কোনো এরর আছে কি না তা দেখতে। |
| `docker exec -it container_id sh` | চলমান কন্টেইনারের ভেতরে ঢুকে কমান্ড রান করতে। |

### **ডকার কম্পোজ (Docker Compose) কমান্ড**

| কমান্ড | কেন ব্যবহার করা হয় |
| --- | --- |
| `docker compose up -d` | কম্পোজ ফাইলের সব সার্ভিস একসাথে চালু করতে। |
| `docker compose up --build -d` | কোড চেঞ্জ করলে পুনরায় বিল্ড করে সব চালু করতে। |
| `docker compose down` | সব সার্ভিস বন্ধ করে কন্টেইনার এবং নেটওয়ার্ক ডিলিট করতে। |
| `docker compose logs -f` | সব সার্ভিসের লগ একসাথে দেখতে। |

## ২. ডকারের গুরুত্বপূর্ণ ফাইলসমূহ

### **১. Dockerfile**

এটি হলো আপনার অ্যাপ্লিকেশনের **রেসিপি**।

* **কাজ:** এটি ডকারকে বলে দেয় যে আপনার অ্যাপ্লিকেশন রান করতে কোন অপারেটিং সিস্টেম লাগবে, কোন প্যাকেজ ইনস্টল করতে হবে এবং কোন কমান্ড দিয়ে অ্যাপটি স্টার্ট হবে।
* **কেন লাগবে:** এটি ব্যবহার করেই আপনি আপনার অ্যাপের নিজস্ব একটি "ইমেজ" তৈরি করেন।

### **২. docker-compose.yml**

এটি হলো **অর্কোস্ট্রেশন ফাইল** বা মাল্টি-সার্ভিস ম্যানেজার।

* **কাজ:** যখন আপনার অ্যাপের সাথে আরও অনেক কিছু লাগে (যেমন: PostgreSQL ডাটাবেস, Redis, বা Prisma Studio), তখন সেগুলোকে একসাথে ম্যানেজ করার জন্য এই ফাইল ব্যবহার করা হয়।
* **কেন লাগবে:** যেন আপনাকে আলাদাভাবে ডাটাবেস বা অ্যাপ রান করতে না হয়; এক কমান্ডেই সব চালু হয়ে যায়।

### **৩. .dockerignore**

এটি অনেকটা `.gitignore`-এর মতো।

* **কাজ:** ইমেজ বিল্ড করার সময় কোন কোন ফাইল বা ফোল্ডার ডকারের ভেতরে যাবে না (যেমন: `node_modules`, `.env`, `.git`), তা এখানে লিখে দেওয়া হয়।
* **কেন লাগবে:** ইমেজের সাইজ ছোট রাখতে এবং সিকিউরিটি বজায় রাখতে।

## ডকার লাইফসাইকেল (সহজ কথায়)

১. আপনি **Dockerfile** লেখেন।
২. আপনি `docker build` দিয়ে একটি **Image** তৈরি করেন।
৩. আপনি `docker run` দিয়ে সেই ইমেজ থেকে একটি **Container** তৈরি করেন।
৪. আপনি **docker-compose.yml** দিয়ে একাধিক কন্টেইনারকে একসাথে নিয়ন্ত্রণ করেন।
